JAIDA 3.0.1 Release Notes

What is Included in the Release

This current release of JAIDA is designed to be used with Java 1.4.0 or later. It will not work with earlier versions of Java.

Check the following list to see which part of AIDA are not fully implemented.

Several JAR files are included in this release. You must include all of the required files in your CLASSPATH to be able to use AIDA. In addition there are several optional files.

Required Files

  • aida.jar - The specification of the AIDA interfaces
  • aida-dev.jar - The developer level specification of the AIDA interfaces
  • freehep-hep.jar - The base (non-graphical) AIDA implementation
  • freehep-base.jar - Some utilities used by freehep-hep
  • openide-lookup.jar - Utility used by AIDA implementation
  • jel.jar - Utility used by AIDA implementation
  • optimizers.jar - Optimization package containing the optimizer UNCMIN

Optional Files

  • JASAIDA.jar - Required if you need to display plots
  • jas.jar - Required if you need to display plots
  • OS-dependent minuitAdapter library - Required to perform fits using Minuit. Configuration:
    • On Windows - add directory with minuitAdapter library file to the PATH
    • On Linux or Solaris - add directory with minuitAdapter library file to the LD_LIBRARY_PATH
    • On Mac OSX - add directory with minuitAdapter library file to the DYLD_LIBRARY_PATH

Limitations/Features of the Current Release

Histograms - We have provided a full implementation of the AIDA 3.0 histogram interfaces, including histogram arithmetic and projections and slices. Slicing of 3D histogram is not yet supported.

Profile - One and two dimensional Profile histograms are fully implemented.

Clouds - We have provided a full implementation of the AIDA 3.0 cloud interfaces. By default clouds will "auto-convert" to histograms when they reach 100,000 entries. You can turn off auto-conversion by specifying the option "autoConvert=no" when creating the Cloud.

DataPointSet - A full implementation of the AIDA 3.0 is implemented including operations among DataPointSets.

Trees and IO - The following methods are not yet implemented, mv, cp, and symlink. The current implementation of  ITree supports reading and writing AIDA standard XML files (.aida files) in either gzip compressed format, or uncompressed format. You can control this via the options parameter when the ITree is created. The default is "compressed=yes".

Tuples - Most of the functionality of AIDA ITuple interface is supported, including creating, filling and querying Tuples. The project methods are implemented, but currently the Evaluators are limited to single columns of the tuple, and filters are not yet supported at all. Full interactive analysis of Tuples stored in .aida files is supported by using the separate JAS Tuple Explorer.

Plotting - Full support for the IPlotter interface is included, including regions and overlays, except that the write() function is not yet implemented. In the Java implementation the Plotter observes all displayed Histograms and Clouds, and will automatically update the display as necessary. Therefore the IPlotter.refresh() method is currently ignored in the Java implementation. Not all AIDA implementations will do this, so you for portability you should still periodically call the refresh() method. Similarly the Java implementation is multi-threaded, so plots will be refreshed while the main program continues to run, and hence the IPlotter.interact() method is also ignored by the Java implementation. Currently only 1D and 2D histograms and clouds can be displayed (not 3D). Histograms with variable bin-widths are not yet displayed correctly.

Fitting - The flexible design of the IFitter interface allows the user to switch very easily the optimizer engine used in the minimization process and the type of fit method used. We currently support two optimizers: Minuit and Uncmin. The available fit methods are: LeastSquares, Chi2, CleverChi2, BinnedMaximumLikelihood and UnbinnedMaximumLikelihood.

Functions- Two types of functions can be created with the IFunctionFactory: built-in functions and scripted functions. Currently the only built-in functions we provide are the Polynomial, the Exponential, the one and two dimensional Gaussian that can be created through the createFunctionByName method of the IFunctionFactory using for the functions model p, e, g and g2 rispectively. No operations among them are currently supported. For more complicated functions the user should create scripted functions throught the createFunctionFromScript method.
